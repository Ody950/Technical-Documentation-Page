Introduction

React is a JavaScript library created by Facebook for building user interfaces. It is not a framework but a tool for building UI components. React creates a virtual DOM in memory and updates the actual DOM only when necessary. This makes it fast and efficient.

Getting Started

You can get started with React by following the instructions on the official React website. You can also use the command-line interface (CLI) tool create-react-app, which expedites the process of developing a React application by installing some packages and creating some files for you. You’ll need to have Node >= 14 on your local development machine (but it’s not required on the server).

Components

React components are independent and reusable bits of code that serve the same purpose as JavaScript functions but work in isolation and return HTML. In React, we mainly have two types of components:

  • 1- Functional Components: Functional components are simply JavaScript functions that return JSX code that tells what should be rendered on the screen.
  • 2- Class Components: Class components are more complex than functional components and are used when we need more features like local state or lifecycle methods.
Props

React Props are like function arguments in JavaScript and attributes in HTML. They are used to pass data from one component to another. To send props into a component, use the same syntax as HTML attributes. 

          
            function Greeting(props) {
              return <h1>Hello, {props.name}!</h1>;
            }
    
            function App() {
              return (
                <div>
                  <Greeting name="Alice" />
                  <Greeting name="Bob" />
                  <Greeting name="Charlie" />
                </div>
              );
            }
State

The state is an object that represents the parts of the app that can change. It is used to store property values that belong to the component. When the state object changes, the component re-renders. The state object is initialized in the constructor.

          
            class Counter extends React.Component {
              constructor(props) {
                super(props);
                this.state = {count: 0};
              }
    
              render() {
                return (
                  <div>
                    <p>You clicked {this.state.count} times</p>
                    <button onClick={() => this.setState({count: this.state.count + 1})}>
                      Click me
                    </button>
                  </div>
                );
              }
            }
Lifecycle Methods

React.js lifecycle methods are used to manage the various stages of a component’s lifecycle, such as mounting, updating, and unmounting. Lifecycle methods can be used to perform actions such as fetching data, setting up subscriptions, or updating the component’s state.

React has four built-in methods that get called when mounting a component: React has four built-in methods that get called when mounting a component: constructor(), static getDerivedStateFromProps(), render(), and componentDidMount(). When updating a component, the following methods are called in this order: static getDerivedStateFromProps(), shouldComponentUpdate(), render(), getSnapshotBeforeUpdate(), componentDidUpdate(). When unmounting a component, componentWillUnmount() is called.

Lifecycle methods can be used to perform actions such as fetching data, setting up subscriptions, or updating the component’s state. They can also be used to clean up resources used by the component when it is removed from the DOM.

Events

React is a JavaScript library for building user interfaces. It is used for handling the view layer for web and mobile apps. React uses a virtual DOM (a JavaScript representation of the real DOM) to manage the state of the application. React also provides a way to handle events in your application.

In React, events are triggered by user interactions such as clicking on a button or hovering over an element. You can use events to update the state of your application and re-render the view.

Here are some examples of events in React:

  • onClick
  • onChange
  • onSubmit
  • onMouseOver
  • onMouseOut

Conditional Rendering

Conditional rendering is a feature in React that allows you to render different components or UI elements based on certain conditions. It is used to change the behavior of an app depending on its state. For instance, you can change the greeting message of your React app to dark during the night. This way you have a different display message depending on the time of day.

In ReactJS, conditional rendering is referred to as Conditional Rendering. To do this, generate a number of components and render them in accordance with certain criteria. There are several ways to use conditional rendering in React. You can use if statements, ternary operators, switch statements, and more.

Lists and Keys

In React, lists are used to display a series of similar items. You can create a list in React by using the map() function to iterate over an array of data and return an array of React elements. Each element in the array should have a unique key prop that helps React identify which items have changed, are added, or are removed. Keys should be given to the elements inside the array to give the elements a stable identity.

Keys allow React to keep track of elements. This way, if an item is updated or removed, only that item will be re-rendered instead of the entire list.

Here’s an example of how you can create a list in React:

        
        const numbers = [1, 2, 3, 4, 5];
        const listItems = numbers.map((number) =>
        <li key={number.toString()}>
            {number}
          
        );
Fragments

Fragments are a common pattern in React where a component returns multiple elements. Fragments let you group a list of children without adding extra nodes to the DOM.

In React, you can use fragments to return multiple elements from a component. The other alternative is to use an HTML element like div to wrap them. Fragments are syntax that allow us to add multiple elements to a React component without wrapping them in an extra DOM node.

Context API

React Context API is a feature in React.js that allows us to pass data through our component trees, giving our components the ability to communicate and share data at different levels. It is safe to use in production with the version 16.3 or latest.

The reason for adding context API is to avoid the passing of props if there is a chain of children components.

Here are the four steps to using React context :

  • 1. Create context using the createContext method.
  • 2. Take your created context and wrap the context provider around your component tree.
  • 3. Use the useContext hook to consume the context.
  • 4. Pass data through the context provider.
Hooks

React Hooks are a feature in React that allow you use state and other React features without writing classes. Hooks were added to React in version 16.8. Because of this, class components are generally no longer needed. Hooks let you use different React features from your components. You can either use the built-in Hooks or combine them to build your own.

There are several built-in hooks in React such as useState, useEffect, useContext, useReducer, useCallback, useMemo, useRef, and useImperativeHandle.

UseState is a Hook that lets you add state to your functional components. It takes an initial value as an argument and returns an array with two items: the current state value and a function that lets you update it. useEffect is another Hook that lets you perform side effects in your functional components. It takes two arguments: a function that performs the side effect and an array of dependencies that tells React when to re-run the effect.

      
        import React, { useState } from 'react';

function Example() {
  // Declare a new state variable, which we'll call "count"
  const [count, setCount] = useState(0);

  return (
    <div>
    <p>You clicked {count} times</p>
    <button onClick={() => setCount(count + 1)}>
        Click me
      
    );}
